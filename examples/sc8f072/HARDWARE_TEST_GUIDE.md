## 🎯 测试目标

### 主要目标
验证 SafeTimer 在 SC8F072 实际硬件上的：
1. **功能正确性** - 定时器按预期工作
2. **资源占用** - RAM/Flash 符合设计目标
3. **稳定性** - 长时间运行无崩溃
4. **时间精度** - 定时误差在可接受范围内
5. **API 可用性** - 所有公开 API 正常工作

### 次要目标
1. 验证 BSP 移植的正确性
2. 验证文档的准确性和完整性
3. 收集实际性能数据

## 🔧 测试环境准备

### 1. 硬件准备

**必需硬件：**
- [ ] SC8F072 开发板或最小系统
- [ ] 3 个 LED（红、绿、蓝推荐）
- [ ] 3 个 330Ω 限流电阻
- [ ] 中微烧录器或兼容编程器（推荐 CMS-WRITER8 LITE）
- [ ] 5V 电源（USB 供电或独立电源）
- [ ] 杜邦线若干

**可选硬件：**
- [ ] 示波器（用于精度测试）
- [ ] 逻辑分析仪（用于时序分析）
- [ ] 万用表（用于电压测量）

**硬件连接图：**
```
SC8F072 最小系统
┌─────────────────────┐
│  SC8F072            │
│                     │
│  P1.0 ──┬─[330Ω]─── LED1 ─── GND
│         │
│  P1.1 ──┬─[330Ω]─── LED2 ─── GND
│         │
│  P1.2 ──┬─[330Ω]─── LED3 ─── GND
│                     │
│  VCC ───────────── 5V
│  GND ───────────── GND
└─────────────────────┘
```

### 2. 软件准备

**编译环境：**
- **SCMCU IDE** v2.00.16 或更高版本（[中微半导体官网下载](https://www.mcu.com.cn/)）
  - 支持 Windows 7/10/11
  - 集成开发环境，包含编译器和调试器
  - 下载地址：https://www.mcu.com.cn/Uploads/download/2023/05/10/SCMCU_IDE_V2.00.16.rar

**烧录工具：**
- **SCMCU Writer** v9.01.15 或更高版本（推荐）
  - 官方烧录软件，支持所有中微 MCU
  - 下载地址：https://www.mcu.com.cn/Uploads/download/2024/04/09/SCMCU_Writer_V9.01.15.zip
- **CMS-WRITER8 LITE**（备选）
  - 通用型烧录器，硬件+软件组合

**可选工具：**
- Proteus 8+ （仿真测试）
- minicom / PuTTY（串口调试）

**⚠️ 注意：** SC8F072 是中微半导体的 RISC 核心 MCU，**不兼容** 8051，**不能使用 SDCC 或其他 8051 编译器**。

### 步骤 3: 硬件连接

**连接顺序：**
```
1. 关闭电源
2. 连接 LED1 到 P1.0（通过 330Ω 电阻到 GND）
3. 连接 LED2 到 P1.1（通过 330Ω 电阻到 GND）
4. 连接 LED3 到 P1.2（通过 330Ω 电阻到 GND）
5. 连接 USB-TTL 编程器
6. 检查接线无误后上电
```

**⚠️ 注意事项：**
- 确认 LED 极性（长脚为正极）
- 确认电阻连接（避免 LED 过流）
- 使用共阴极接法（LED 负极接 GND）

---

### 步骤 4: 烧录固件

**方法 A: 使用 SCMCU Writer（Windows 推荐）**
```
1. 打开 SCMCU Writer 软件
2. 连接设置：
   - MCU 选择：SC8F072 或 SC8F 系列
   - 通信方式：选择 "UART" 或 "USB"（取决于烧录器）
   - 波特率：115200（默认）

3. 加载程序：
   - 点击 "Open File" 按钮
   - 选择编译生成的 .hex 文件

4. 烧录选项：
   - IRC 频率：8MHz（默认）
   - 看门狗：根据需要选择
   - 上电复位：建议启用

5. 开始烧录：
   - 点击 "Download" 按钮
   - 给 SC8F072 上电（或按复位键）
   - 等待自动烧录完成（约 5-10 秒）
   - 显示 "Download Success" 表示成功

6. 断开连接，观察 LED 是否开始闪烁
```

**方法 B: 使用 CMS-WRITER8 LITE（硬件烧录器）**
```
1. 连接硬件烧录器到 SC8F072 的 ICSP 接口
2. 打开配套软件
3. 选择芯片型号：SC8F072
4. 加载 .hex 文件
5. 点击 "Program" 按钮
6. 等待烧录完成
```

**方法 C: 使用 SCMCU IDE 内置下载（如果支持）**
```
1. 在 SCMCU IDE 中，Project → Download
2. 配置下载器类型和接口
3. 点击 "Download" 开始烧录
```

**✅ 验证点：**
- [ ] 烧录成功无错误
- [ ] 烧录后自动复位
- [ ] LED 立即开始闪烁

## 🔍 问题排查指南

### 问题 1: 编译失败

**症状：** 在 SCMCU IDE 中编译报错

**可能原因和解决方案：**

1. **SCMCU IDE 未正确安装**
   ```
   解决方案：
   1. 从中微半导体官网下载最新版 SCMCU IDE
   2. 以管理员身份运行安装程序
   3. 安装后重启计算机
   4. 验证：File → New Project 能否正常打开
   ```

2. **项目文件路径包含中文或空格**
   ```
   解决方案：
   1. 将项目移动到纯英文路径（如 C:\Projects\SafeTimer\）
   2. 确保路径中没有空格
   3. 重新打开项目并编译
   ```

3. **头文件路径配置错误**
   ```
   解决方案：
   1. Project → Options → C/C++ → Include Paths
   2. 添加：../../include
   3. 确认所有路径都存在
   ```

---

### 问题 2: 烧录失败

**症状：** SCMCU Writer 提示"无法连接"或"超时"

**可能原因和解决方案：**

1. **串口未识别**
   ```
   解决方案：
   - 在设备管理器中检查 COM 口是否正常
   - 确认驱动程序已安装（CH340/CP2102）
   - 尝试更换 USB 口
   ```

2. **波特率不匹配**
   ```
   解决方案：
   - 在 SCMCU Writer 中尝试不同波特率
   - 推荐值：115200、57600、19200
   ```

3. **上电时序错误**
   ```
   解决方案：
   - 先点击 SCMCU Writer 的 "Download" 按钮
   - 再给 SC8F072 上电（或按复位键）
   - 应在 2-3 秒内开始下载
   ```

4. **烧录器接线错误**
   ```
   正确接线（UART 模式）：
   烧录器 TXD  →  SC8F072 RXD (P3.0)
   烧录器 RXD  ←  SC8F072 TXD (P3.1)
   烧录器 GND  —  SC8F072 GND
   烧录器 VCC  →  SC8F072 VCC（可选，建议独立供电）

   ⚠️ 注意：TXD 和 RXD 交叉连接！
   ```

---

### 问题 3: LED 不闪烁

**症状：** 烧录成功但 LED 无反应

**可能原因和解决方案：**

1. **LED 极性反接**
   - 确认长脚（正极）连接到 P1.x
   - 短脚（负极）通过电阻连接到 GND

2. **LED 损坏**
   - 用万用表测试 LED
   - 更换新 LED

3. **GPIO 配置错误**
   - 检查 `main.c` 中 `P1` 的配置
   - 确认引脚设置为输出模式

4. **程序未运行**
   ```c
   // 在 main.c 开头添加测试代码
   P1_0 = 1; // 点亮 LED1 测试
   while(1); // 死循环，如果 LED1 亮则说明程序运行
   ```

---

### 问题 4: LED 闪烁周期不正确

**症状：** LED 闪烁，但周期明显偏离预期

**可能原因和解决方案：**

1. **晶振频率不匹配**
   - 检查实际晶振频率（可能是 11.0592MHz 而不是 8MHz）
   - 修改 `bsp_sc8f072.c` 中的 Timer0 重装载值
   ```c
   // 对于 11.0592MHz：
   #define TIMER0_RELOAD_H  0xFC
   #define TIMER0_RELOAD_L  0x18
   ```

2. **定时器配置错误**
   - 检查 Timer0 模式设置
   - 确认预分频器配置正确（1:12）

3. **`safetimer_process()` 调用频率不足**
   - 确保主循环中正确调用
   - 检查是否有长耗时操作阻塞主循环

---

### 问题 5: 运行一段时间后死机

**症状：** LED 闪烁正常，但几分钟/小时后停止

**可能原因和解决方案：**

1. **栈溢出**
   ```bash
   # 检查栈使用情况
   grep "Stack" build/safetimer_sc8f072.mem

   # 如果接近上限，减少 MAX_TIMERS
   # 编辑 ../../include/safetimer_config.h
   #define MAX_TIMERS 2  // 降低到 2
   ```

2. **看门狗复位**
   ```c
   // 在主循环添加看门狗喂狗
   void main(void) {
       while(1) {
           safetimer_process();
           WDT_CONTR |= 0x10;  // 喂狗（SC8F072）
       }
   }
   ```

3. **溢出处理错误**
   - 验证 `bsp_get_ticks()` 正确实现
   - 确保 `s_system_ticks` 是 `volatile`

---

### 问题 6: 编译警告处理

**常见警告和处理：**

1. **"unused variable"（未使用的变量）**
   ```
   原因：回调函数的 user_data 参数未使用
   解决：可忽略，或添加 (void)user_data; 消除警告
   ```

2. **"implicit return type"（隐式返回类型）**
   ```
   原因：函数声明有返回值但未 return
   解决：检查代码，确保所有分支都有返回值
   ```

3. **"undefined identifier"（未定义的标识符）**
   ```
   原因：函数或变量未声明就使用
   解决：添加 #include 或变量/函数声明
   ```

**⚠️ 注意：** SCMCU IDE 的警告信息可能与其他编译器不同，应重点关注红色错误信息。
